●標準(tribute_c.mac)との違い
	・タグづけにctagsを使用している
	・クラスの定義でインライン関数があってもちゃんとクラスのメンバと認識する
	・オプションでファイルの先頭を示す項目を追加可能


●制限事項
	・ctagsの制限のため、Shift-JIS/ASCII/EUC-JP/UTF8以外のエンコードのソースファ
	　イルは正しく扱えない可能性があります。
	・ダイアログの設定でソートをするようにしている場合で、対象ファイルにクラスの
	　定義と一般の関数が混在している場合、クラスの関数をまとめる親のクラスの項目
	　が正しい位置に設定されない事があります。ただし、正しくないのは見た目の位置
	　だけで、動作そのものには影響がありません。
	・クラス内にクラスを定義した場合には内部のクラスは外側のクラスの子にならずに
	　「外クラス::内クラス」という形式で親として登録されます。


●インストール/設定
▼マクロ本体について
	・「tribute\plugin」の中に「ctags_c.mac」を入れてください。
	・「tribute\tribute.ini」を使いたいところだけ変更
	　※デフォルトの「tribute_c.mac」と置き換えるなら「[ALIAS]」セクションの中の
	　　「C=tribute_c.mac」を「C=ctags_c.mac」に書き換えてください。
	・オプションの設定は「tribute\plugin\plugin.ini」の「[C]」のセクションです。
	・ファイル先頭を示す項目を追加したい場合は「DlgHead=yes」を追記してください。

▼ctagsについて
	　ctags.exe(日本語対応版)を「http://hp.vector.co.jp/authors/VA025040/ctags/」
	あたりから持ってきてください。
	※2004/10/24現在
	http://hp.vector.co.jp/authors/VA025040/ctags/downloads/ec553j1w32.zip
	です。
	解凍するといろいろなファイルに混ざって「ctags.exe」というファイルが出ます。
	これをパスの通った場所に移せばOKです。残りは不要です。
	パスの通った所に置くことが出来ない場合は、前述のplugin.iniのCセクションに、
	「ctags=ctags.exeのフルパス」のように記述してください。
